claimpipe

A cloud-agnostic, event-sourced claim-ingestion pipeline. A PDF + JSON claim is logged → run through OCR → scored by multiple LLM models (tiered routing) → persisted → the customer is notified by webhook. Same code on a laptop and on any cloud.

Architecture

flowchart LR
    Cust([Customer])
    API[FastAPI ingestion]
    subgraph ORCH[Temporal · decider + state machine]
      WF[ClaimWorkflow]
      OCR[OCR activity]
      LLM[LLM tiered routing]
      AGENT[LangGraph review agent]
    end
    EV[(claim_events source of truth)]
    PROJ[(projections status · model_outputs)]
    BUS[(Kafka / Redpanda)]
    NOTIF[Notifier]
    Cust -->|POST /claims + PDF| API
    API -->|CLAIM_RECEIVED| EV
    API -->|start| WF
    WF --> OCR --> LLM
    LLM -->|escalate| AGENT
    WF -->|emit events| EV
    EV --> PROJ
    EV -->|outbox| BUS --> NOTIF
    NOTIF -->|HMAC webhook| Cust

The Temporal workflow decides and emits domain events; claim_events is the append-only source of truth; status & model_outputs are projections. Kafka is the fan-out bus (not the record), fed by a transactional outbox so no event is lost.

Pipeline lifecycle

RECEIVED→ OCR_RUNNING→ OCR_DONE→ LLM_RUNNING→ LLM_DONE→ PERSISTED→ NOTIFIED

Transitions are validated in one place; illegal jumps raise. Terminal branches: FAILED, PARTIAL_SUCCESS, NOTIFY_FAILED (with DLQ replay back to NOTIFIED).

What's inside

⏱️ Durable orchestration

Self-hosted Temporal is the decider & state machine — durable retries, timeouts, and event-driven dormancy gates (no polling).

🧾 Event-sourced

Append-only claim_events is the source of truth; status & model_outputs are projections folded from it — audit-grade by construction.

📡 Kafka fan-out

Transactional outbox → relay → Kafka/Redpanda decouples consumers (notifier, analytics) — the bus is the pipe, not the record.

🧠 Tiered LLM routing

Cost-tier model first; escalate on low confidence or high-value claims. Structured output for schema-guaranteed results.

🕸️ LangGraph agent

On escalation, a extract → validate → recommend → critic graph runs inside a Temporal activity; a human owns the final call.

🔌 Adapters everywhere

OCR, object store, message bus, and LLM providers behind Protocols — swap cloud/provider by config, not a rewrite.

🚚 One image, four roles

api / worker / relay / notifier from a single container; Helm chart + Terraform deploy to any Kubernetes.

🧪 Hermetic CI

33 tests run with Temporal's in-process test env — no Docker needed. Local = prod parity.
